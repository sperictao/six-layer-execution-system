  - 每个 activity 拥有独立目录，包含 card.md + 六层资源文件

  - 旧 tasks 单文件顺便拆成新格式

  - plugins/six-layer-execution-system/ 内所有文件重组
  - ACTIVE.md 格式 v2→v3
  - ~12 个 checker/脚本路径适配
  - ~6 个规范/文档同步更新
  - Hermes skill 副本同步
- constraints:
  - ACTIVE.md focus-first 语义不变
  - 恢复链必须可工作
  - 不丢失任何现有数据
  - 旧文件保留（归档而非删除）
- non_goals:
  - 不改变六层系统的执行语义（仍 demand→contract→roadmap→tasks→ACTIVE）
